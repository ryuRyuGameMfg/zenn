## はじめに

Claude Codeに「このAPIの処理フローを教えて」と聞くと、どうなるか。

デフォルトだと、長いテキストが返ってきます。「まずリクエストを受け取り、次にバリデーションを行い、その後DBに保存し...」と延々と続く文章。構造を把握するのに何度も読み返す必要がある。

ところが、CLAUDE.mdに **「デフォルト出力はASCII図解」と1行書くだけで、出力が劇的に変わります**。

**Before:**

```text
処理フローは以下の通りです。まずユーザーからのリクエストを受け取り、
バリデーションを実行します。バリデーションが成功した場合はDBに保存し、
失敗した場合はエラーレスポンスを返します。保存が完了したら...（以下省略）
```

**After:**

```text
┌─────────┐    ┌──────────┐    ┌─────────┐
│ リクエスト │───→│バリデーション│───→│ DB保存   │
└─────────┘    └──────────┘    └─────────┘
                    │
                    ↓ エラー時
               ┌──────────┐
               │ エラー応答  │
               └──────────┘
```

一目で構造がわかる。この記事では、CLAUDE.mdにASCII図ルールを定義して出力を制御する方法を解説します。

## なぜMermaidではなくASCII図なのか

「図解ならMermaidでよくない?」と思うかもしれません。僕も最初はMermaidを指定していました。しかし、Claude Codeという環境では **ASCII図の方が圧倒的に実用的** だと気づいたのです。

理由は4つあります。

**1. ターミナル上でそのまま読める**

Claude CodeはCLIツールです。素のターミナルではMermaidコードがそのまま表示され、図として読めません。ASCII図ならターミナルにそのまま表示されます。

**2. コピペがそのままできる**

Slackに貼り付けるとき、ASCII図はコードブロックに入れるだけ。MermaidはSlack等ではコードのまま表示されます。

**3. 80文字幅でターミナル互換**

幅制限を設けることで、どのターミナルでも崩れずに表示されます。

**4. 生成が高速**

テキストベースなので、Claudeの生成速度が速い。複雑なMermaid構文を組み立てる必要がありません。

:::message
Mermaidは記事やドキュメントに埋め込む場合に最適です。ターミナルで即座に確認したい場合はASCII図が勝ります。用途に応じて使い分けましょう。
:::

| 観点 | ASCII図 | Mermaid |
|------|--------|---------|
| ターミナル表示 | そのまま読める | レンダリング必要 |
| コピペ | そのまま | コード必要 |
| 表現力 | 限定的 | 豊富 |
| 生成速度 | 高速 | やや遅い |
| 最適な場面 | CLIの即座の回答 | ドキュメント・記事 |

## 7種類のASCII図テンプレート

CLAUDE.mdに定義している7種類の図解タイプを紹介します。これを定義しておくと、 **Claudeが質問内容に応じて最適な図解を自動選択** してくれます。

### 1. フローチャート（処理フロー・手順）

処理の流れや手順を表現するときに使います。分岐も矢印で直感的に表現できます。

```text
┌─────────┐    ┌──────────┐    ┌─────────┐
│ ユーザー入力 │───→│ バリデーション │───→│ DB保存   │
└─────────┘    └──────────┘    └─────────┘
                    │
                    ↓ エラー時
               ┌──────────┐
               │ エラー表示  │
               └──────────┘
```

### 2. ステート図（状態遷移）

ゲーム開発やUIの状態管理など、状態の遷移を表現するときに最適です。

```text
┌────────┐  start  ┌────────┐  play   ┌────────┐
│  Title  │────────→│  Menu   │────────→│ InGame  │
└────────┘         └────────┘         └────────┘
                        ↑                  │
                        │    game over     │
                        └──────────────────┘
```

### 3. シーケンス図（時系列のやり取り）

API通信やクライアント・サーバー間のやり取りを時系列で表現します。

```text
Client          Server          Database
  │               │               │
  │──── GET ──────→│               │
  │               │── SELECT ────→│
  │               │←── rows ──────│
  │←── JSON ──────│               │
  │               │               │
```

### 4. ツリー図（階層構造・分類）

ディレクトリ構造やカテゴリ分類など、親子関係の表現に使います。

```text
project/
├── src/
│   ├── components/
│   │   ├── Header.tsx
│   │   └── Footer.tsx
│   └── pages/
│       └── index.tsx
├── tests/
└── package.json
```

### 5. 表形式（比較・選択肢）

3項目以上の比較は表形式を使います。これは通常のMarkdownテーブルで表現するため、ASCII罫線は不要です。

### 6. ER図風（関係性・データ構造）

エンティティ間の関係を表現します。DB設計の議論で重宝します。

```text
┌──────────┐       ┌──────────┐
│   User    │ 1───N │  Order   │
├──────────┤       ├──────────┤
│ id        │       │ id       │
│ name      │       │ user_id  │
│ email     │       │ total    │
└──────────┘       └──────────┘
```

### 7. 矢印付きボックス図（依存関係・影響範囲）

モジュール間の依存関係やインポート構造を表現します。

```text
┌──────────┐
│   App     │
├──────────┤
│  imports  │
└────┬─────┘
     │
     ├──→ ┌──────────┐
     │    │  Router   │
     │    └──────────┘
     │
     └──→ ┌──────────┐
          │  Store    │
          └──────────┘
```

:::message
7種類すべてを覚える必要はありません。CLAUDE.mdに「図解タイプの使い分け」を定義しておけば、 **Claudeが質問の種類から最適な図を自動判断** します。
:::

## CLAUDE.mdへの設定方法

実際にCLAUDE.mdに記述する内容を紹介します。必要なのは2つのセクションだけです。

### 出力フォーマットの定義

まず、デフォルトの出力形式をASCII図に指定します。

```markdown:CLAUDE.md
## 出力フォーマット
- デフォルト: ASCII図解（```text ブロック）+ 補足テキスト（数行以内）
- テキスト詳細展開: ユーザーが「詳しく」と求めた場合のみ
- 3項目以上の比較は表形式必須
- 禁止: Bash経由の図表生成（折りたたまれるため）
```

**「デフォルト: ASCII図解」の1行が最も重要です**。これにより、Claudeは毎回の回答で自動的に図解を優先するようになります。「詳しく」と言えばテキスト展開に切り替わるため、柔軟性も確保できます。

### 図解タイプと描画ルールの定義

次に、7種類の図解マッピングと描画ルールを記述します。

```markdown:CLAUDE.md
### 図解タイプの使い分け
- 処理フロー・手順 → フローチャート
- 比較・選択肢 → 表形式
- 状態遷移・ステータス変化 → ステート図
- 時系列・やり取り → シーケンス図
- 階層構造・分類 → ツリー図
- 関係性・データ構造 → ER図風
- 依存関係・影響範囲 → 矢印付きボックス図

### ASCII図の描画ルール
- 罫線文字を使用: ┌ ┐ └ ┘ ─ │ ├ ┤ ┬ ┴ ┼ →
- 必ず ```text ブロックで囲む
- 幅は80文字以内（ターミナル互換）
- ボックス内テキストは簡潔に（1行推奨）
```

:::message
「罫線文字を使用」の指定がないと、Claudeは`+`や`-`や`|`でASCII図を描くことがあります。罫線文字を明示することで、 **統一感のある見やすい図** が生成されます。
:::

### Before / Afterの実例

この設定を入れる前と後で、同じ質問への回答がどう変わるかを示します。

**質問**: 「認証フローを説明して」

**Before（ルールなし）:**

Claudeが5段落のテキストで回答。認証フローの各ステップを文章で説明。全体像の把握に時間がかかる。

**After（ルールあり）:**

```text
┌────────┐    ┌────────┐    ┌────────┐
│ ログイン  │───→│ JWT発行  │───→│ 認証完了  │
└────────┘    └────────┘    └────────┘
     │                          │
     ↓ 失敗                      ↓
┌────────┐               ┌────────┐
│ エラー表示 │               │ API許可  │
└────────┘               └────────┘
```

認証フローの全体像が一瞬で把握できる。補足が必要なら「詳しく」と言えばテキストで展開されます。

## まとめ

CLAUDE.mdに「出力フォーマット」と「ASCII図ルール」を定義するだけで、Claude Codeの出力品質が劇的に変わります。

**設定のポイント:**

1. **デフォルト出力をASCII図に指定する** -- これだけで長文テキストが構造化された図解に変わる
2. **7種類の図解マッピングを定義する** -- Claudeが質問に応じて最適な図を自動選択する
3. **描画ルールで罫線文字と幅を指定する** -- 統一感のある読みやすい図が生成される
4. **MermaidとASCII図を使い分ける** -- ドキュメントにはMermaid、ターミナルにはASCII

:::message
この設定はCLAUDE.mdに約20行を追加するだけです。トークン消費への影響は最小限で、 **出力の可読性は大幅に向上** します。
:::

テキストの壁に悩んでいる方は、まずCLAUDE.mdに「デフォルト: ASCII図解」の1行を追加してみてください。たった1行で、Claude Codeとの対話体験が変わるはずです。
